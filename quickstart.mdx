# Quickstart

Let's build a complete user API with authentication, demonstrating NduloJS's Result pattern, dependency injection, and Clean Architecture in action.

<Note>
  This guide uses real code from the NduloJS starter app example. You'll learn the core patterns by building a working feature.
</Note>

## Create your app

<Steps>

<Step title="Initialize the project">

Create a new NduloJS application:

```bash
npx ndulojs create my-api
cd my-api
bun install
```

This scaffolds a complete project with Clean Architecture structure.

</Step>

<Step title="Understand the structure">

Your project now has this structure:

```
my-api/
├── src/
│   ├── modules/
│   │   └── users/
│   │       ├── application/
│   │       │   ├── dtos/user.dto.ts
│   │       │   ├── ports/user.port.ts
│   │       │   └── services/user.service.ts
│   │       └── infrastructure/
│   │           ├── http/controllers/user.controller.ts
│   │           └── persistence/user.repository.ts
│   └── index.ts
└── package.json
```

<Tip>
  Each module is self-contained with its own application logic (services, DTOs, ports) and infrastructure (controllers, repositories).
</Tip>

</Step>

<Step title="Explore the entry point">

Open `src/index.ts` to see how everything wires together:

```typescript src/index.ts
import { createApp, createContainer, Ok } from 'ndulojs';
import { createUserRepository } from './modules/users/infrastructure/persistence/user.repository.js';
import { createUserService } from './modules/users/application/services/user.service.js';
import { createUserController } from './modules/users/infrastructure/http/controllers/user.controller.js';

const PORT = Number(process.env['PORT']) || 3000;

// Set up dependency injection
const container = createContainer()
  .register('UserRepository', () => createUserRepository())
  .register('UserService',    (c) => createUserService(c.resolve('UserRepository')));

// Create HTTP app
const app = await createApp({
  port: PORT,
  cors: { origins: '*' },
  logger: { enabled: true, level: 'info' },
});

// Health check endpoint
app.get('/health', () => Ok({ status: 'ok' }));

// Register user routes
createUserController(app, container.resolve('UserService'));

app.listen(PORT);
```

Key concepts:
- **Container**: Manages dependencies with type-safe resolution
- **Factories**: Pure functions that create service instances
- **Result pattern**: The health check returns `Ok(value)` instead of throwing

</Step>

</Steps>

## Understanding the Result pattern

Let's examine how the user service handles errors with typed Results:

```typescript user.service.ts
import { Ok, Err, ErrorFactory } from 'ndulojs';
import type { Result, AppError } from 'ndulojs';

export const createUserService = (repo: IUserRepository): IUserService => ({
  async register({ email, password, name }: RegisterDTO): Promise<Result<AuthResponseDTO, AppError>> {
    // Check if user already exists
    const existing = await repo.findByEmail(email);
    if (existing) return Err(ErrorFactory.conflict('Email already in use', 'email'));

    // Create new user
    const passwordHash = await hash(password);
    const user = await repo.create({ email, password, name, passwordHash });
    
    // Return success with token
    return Ok({ user, token: signToken(user.id) });
  },

  async login({ email, password }: LoginDTO): Promise<Result<AuthResponseDTO, AppError>> {
    const record = await repo.findByEmail(email);
    if (!record) return Err(ErrorFactory.unauthorized('Invalid credentials', 'invalid_credentials'));

    const valid = await verify(record.passwordHash, password);
    if (!valid) return Err(ErrorFactory.unauthorized('Invalid credentials', 'invalid_credentials'));

    const { passwordHash: _, ...user } = record;
    return Ok({ user, token: signToken(user.id) });
  },

  async me(userId: string): Promise<Result<UserDTO, AppError>> {
    const user = await repo.findById(userId);
    if (!user) return Err(ErrorFactory.notFound('User not found', 'User', userId));
    return Ok(user);
  },
});
```

<Warning>
  Notice there's no try/catch. Errors are explicit return values using `Err(ErrorFactory.xxx())`. The framework automatically maps these to HTTP status codes.
</Warning>

Error mapping:
- `ErrorFactory.conflict()` → 409 Conflict
- `ErrorFactory.unauthorized()` → 401 Unauthorized  
- `ErrorFactory.notFound()` → 404 Not Found

## The controller layer

Controllers receive the HTTP adapter and services, then define routes:

```typescript user.controller.ts
import { Err, ErrorFactory } from 'ndulojs';
import type { IHttpAdapter } from 'ndulojs';
import type { IUserService } from '../../../application/ports/user.port.js';
import { verifyToken } from '../../../application/services/user.service.js';

export const createUserController = (app: IHttpAdapter, service: IUserService): void => {
  app.group('/users', (r) => {
    r.post('/register', async ({ body }) => {
      const { email, password, name } = body as { email: string; password: string; name: string };
      return service.register({ email, password, name });
    });

    r.post('/login', async ({ body }) => {
      const { email, password } = body as { email: string; password: string };
      return service.login({ email, password });
    });

    r.get('/me', async ({ headers }) => {
      const token = headers['authorization']?.replace('Bearer ', '');
      if (!token) return Err(ErrorFactory.unauthorized('Missing token', 'missing_token'));

      const userId = verifyToken(token);
      if (!userId) return Err(ErrorFactory.unauthorized('Invalid token', 'invalid_token'));

      return service.me(userId);
    });
  });
};
```

<Tip>
  Controllers are thin — they just parse requests and call services. All business logic lives in the service layer.
</Tip>

## Defining ports (interfaces)

Ports define contracts between layers:

```typescript user.port.ts
import type { Result, AppError } from 'ndulojs';
import { AuthResponseDTO, LoginDTO, RegisterDTO, UserDTO } from '../dtos/user.dto';

export interface IUserRepository {
  findByEmail(email: string): Promise<(UserDTO & { passwordHash: string }) | null>;
  findById(id: string): Promise<UserDTO | null>;
  create(data: RegisterDTO & { passwordHash: string }): Promise<UserDTO>;
}

export interface IUserService {
  register(data: RegisterDTO): Promise<Result<AuthResponseDTO, AppError>>;
  login(data: LoginDTO): Promise<Result<AuthResponseDTO, AppError>>;
  me(userId: string): Promise<Result<UserDTO, AppError>>;
}
```

Notice that services return `Result<T, AppError>` — this makes error handling explicit in the type system.

## Add a new module

Let's add a new feature using the CLI:

```bash
ndulo generate module posts
```

This creates:

```
src/modules/posts/
├── events/post.events.ts
├── application/
│   ├── dtos/post.dto.ts
│   ├── ports/post.port.ts
│   └── services/post.service.ts
└── infrastructure/
    ├── persistence/post.repository.ts
    └── http/controllers/post.controller.ts
```

<Tip>
  The CLI scaffolds the complete Clean Architecture structure. You just fill in the business logic.
</Tip>

## Key takeaways

You've learned:

1. **Result pattern**: Return `Ok(value)` or `Err(error)` instead of throwing exceptions
2. **Dependency injection**: Use `createContainer()` to manage dependencies with type safety
3. **Clean Architecture**: Separate application logic (services) from infrastructure (controllers, repositories)
4. **Error handling**: Use `ErrorFactory` to create typed errors that automatically map to HTTP status codes
5. **CLI scaffolding**: Generate modules with `ndulo generate module <name>`
