---
title: ndulojs create
description: Scaffold a new NduloJS project with the full structure
---

The `ndulojs create` command scaffolds a new project with the complete NduloJS structure, including configuration files, entry point, and directory layout.

## Usage

```bash
npx ndulojs create <name>
```

### Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `<name>` | Yes | Project name and directory name |

<Note>
The command will fail if a directory with the same name already exists in the current location.
</Note>

## Examples

### Create a new project

```bash
npx ndulojs create my-app
```

**Output:**
```bash
✓ Project "my-app" ready

Next steps:
  cd my-app
  bun install
  cp .env.example .env
  bun dev
```

### Generated structure

```
my-app/
├── src/
│   ├── modules/          # Your business modules go here
│   └── index.ts          # Application entry point
├── package.json
├── tsconfig.json
├── .env.example
└── .gitignore
```

## Next steps

<Steps>
  <Step title="Navigate to your project">
    ```bash
    cd my-app
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>
      ```bash Bun
      bun install
      ```

      ```bash npm
      npm install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure environment">
    ```bash
    cp .env.example .env
    ```

    Edit `.env` to customize your port and environment settings.
  </Step>

  <Step title="Start development server">
    ```bash
    bun dev
    ```

    Your app will be available at `http://localhost:3000/health`
  </Step>

  <Step title="Generate your first module">
    ```bash
    npx ndulo generate module users
    ```

    See [ndulo generate](/cli/generate) for module generation details.
  </Step>
</Steps>

## Error handling

### Directory already exists

If the target directory exists:

```bash
npx ndulojs create my-app
```

**Output:**
```bash
✗ Directory "my-app" already exists.
```

<Tip>
Choose a different name or remove the existing directory before running the command.
</Tip>
